# Domain Problem

## 1. 문제 배경

금융 거래 이벤트는 짧은 시간에 대량으로 발생할 수 있습니다. 거래 이벤트에는 결제, 이체, 출금, 고액 거래, 새 기기 거래, 해외 접속 거래 등이 포함됩니다.

이상거래 탐지가 거래 저장 이후의 배치 처리에만 의존하면 위험 거래를 사후에 발견하게 됩니다. 따라서 거래 이벤트를 실시간으로 수집하고, 사용자별 최근 거래 패턴을 기준으로 위험도를 계산하는 구조가 필요합니다.

## 2. 이 시스템이 다루는 문제

- 대량 거래 이벤트 유입
- 사용자별 거래 순서 보장
- 실시간 이상거래 탐지 지연
- Consumer 장애 시 이벤트 재처리
- Redis 장애 시 탐지 정확도 저하
- 실패 이벤트의 DLQ 보관과 수동 재처리
- 처리 결과와 감사 로그 저장

## 3. 해결하지 않으면 발생하는 문제

- 위험 거래 탐지 지연
- 동일 이벤트 재처리 시 중복 탐지 결과 생성
- Consumer 장애 이후 이벤트 누락
- 실패 이벤트 원인 추적 불가
- 운영자가 DLQ 이벤트를 확인하거나 복구할 수 없음
- API 응답 시간만 보고 비동기 처리 지연을 놓침

## 4. 범위

이 프로젝트는 실제 금융 원장을 구현하지 않습니다. 거래 승인 여부를 결정하는 계정계 시스템도 구현하지 않습니다.

이 프로젝트의 범위는 거래 이벤트가 이미 발생했다는 전제에서, 해당 이벤트를 Kafka로 수집하고 이상거래 탐지 결과를 생성하는 실시간 이벤트 처리 파이프라인입니다.
