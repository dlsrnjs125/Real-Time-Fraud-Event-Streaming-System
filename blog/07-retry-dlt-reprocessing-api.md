# Retry/DLT와 운영자 재처리 API 설계

> Status: Draft
> 이 글은 구현과 측정 결과가 추가되면서 갱신됩니다.

## 문제

모든 실패 이벤트를 무한 재시도하면 장애 원인을 가리고 시스템 부하를 키울 수 있습니다.

## 초기 설계

일시 실패는 retry topic으로 보내고, 처리 불가능한 이벤트는 DLT로 보관합니다.

## 구현

운영자 API로 DLQ 이벤트를 조회, 재처리, 폐기할 수 있게 합니다.

## 측정 또는 재현

잘못된 payload를 DLT로 보내고 재처리 시 중복 FraudResult가 생기지 않는지 확인합니다.
